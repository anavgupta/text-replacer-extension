Section 01

Extension Architecture

A Manifest V3 Chrome/Edge extension has three isolated execution contexts that communicate through Chrome APIs. No component can directly call functions in another.

Content Script

content.js · 176 lines
  • Injected into every web page
  • Walks the DOM tree, replaces text
  • Watches for dynamic content changes
  • Scoped by URL pattern matching
  • Runs in page context (isolated world)

Popup UI

popup.html + popup.js + popup.css · ~1300 lines
  • Appears when user clicks extension icon
  • CRUD for replacement rules
  • Search, filter, toggle, dark mode
  • Import/Export (JSON, CSV)
  • Toast notifications with undo

Background Worker

background.js · 66 lines
  • Service worker (no persistent page)
  • Manages badge count and colors
  • Context menu "Create rule from selection"
  • Listens to storage changes
  • Broadcasts toggle to all tabs

Section 02

DOM Tree Traversal

The content script recursively walks every node in document.body, but only modifies TEXT_NODE elements. This preserves HTML structure, event listeners, and element attributes.

How the DOM looks to our walker

<body>
  <p>
    "Hello " ← TEXT_NODE: we replace here
    <strong>
      "World" ← TEXT_NODE: and here
    </strong>
    "!"
  </p>
  <script>...</script> ← SKIPPED (SKIP_TAGS)
  <style>...</style> ← SKIPPED
  <textarea>...</textarea> ← SKIPPED
</body>
content.jsCore replacement engine
const SKIP_TAGS = new Set(['SCRIPT', 'STYLE', 'TEXTAREA', 'NOSCRIPT', 'CODE', 'PRE']); function replaceTextInNode(node, replacements) { if (node.nodeType === Node.TEXT_NODE) { // Leaf: actual text let text = node.textContent; replacements.forEach(rule => { const regex = new RegExp(pattern, 'gi'); text = text.replace(regex, rule.replacementText); }); if (modified) node.textContent = text; } else if (node.nodeType === Node.ELEMENT_NODE) { // Branch: recurse if (SKIP_TAGS.has(node.tagName)) return; // Skip dangerous tags for (let child of node.childNodes) { replaceTextInNode(child, replacements); } } }

Section 04

Regex Engine & Whole Word

User input is escaped before becoming a regex pattern. Whole-word matching uses \b for alphanumeric terms and whitespace lookaround for terms with special characters like $50.

const escapedText = find.replace(/[.*+?^${}()|[\]\\]/g, '\\$&'); if (wholeWord) { const hasSpecialChars = /[^a-zA-Z0-9_]/.test(find); if (hasSpecialChars) { pattern = `(?<=^|[\\s])${escapedText}(?=[\\s]|$)`; // Whitespace boundaries } else { pattern = `\\b${escapedText}\\b`; // Word boundaries } }
Section 05

MutationObserver Debounce

SPAs fire hundreds of DOM mutations per second. Without debouncing, each triggers a full storage read + DOM walk. The debounce waits 150ms for mutations to settle, then runs once.

Mutation
→
clearTimeout
→
setTimeout(150ms)
→
...quiet...
→
1 DOM Walk
content.js6 lines that cut CPU by 90%+
let _debounceTimer = null; function debouncedReplacements(delay = 150) { clearTimeout(_debounceTimer); // Cancel pending work _debounceTimer = setTimeout( () => performReplacements(), delay // Schedule new ); } // MutationObserver uses debounced version const observer = new MutationObserver(() => { debouncedReplacements(); // NOT performReplacements() });

Section 06

Inter-Component Messaging

The three extension contexts can't share memory. They communicate via chrome.runtime.sendMessage (popup/background) and chrome.tabs.sendMessage (to content scripts).

User toggles global switch in popup

popup.js saves globalEnabled to storage, calls updateBadge()

Popup sends badge update to background

chrome.runtime.sendMessage({ action: 'updateBadge', count })

Background updates the icon badge

chrome.action.setBadgeText({ text: count }) with green or red color

Storage change event fires in content script

chrome.storage.onChanged triggers performReplacements()

Context menu: "Create replacement for ..."

Background stores selected text via _pendingRuleText, popup reads it on open and pre-fills the modal

Section 07

Storage Strategy

Rules persist across sessions via chrome.storage. A helper function tries sync first (cross-device), falls back to local (for Edge without Microsoft account).

content.js / background.jsStorage abstraction
function getStorage() { try { if (chrome.storage.sync) return chrome.storage.sync; } catch (e) { /* sync unavailable in Edge */ } return chrome.storage.local; // Always works }

Section 08

Multi-Format Import/Export

The extension reads three JSON structures and CSV. Array scopes like ["a", "b"] are automatically joined into comma-separated strings.

Standard Backup

{ "replacements": [...] }
  • Exported by this extension
  • Full round-trip fidelity
  • Includes enabled/wholeWord state

Codes Format

{ "codes": [{ code, name, scope }] }
  • code → textToReplace
  • name → replacementText
  • scope can be string or array

CSV

textToReplace,replacementText,scope,...
  • Standard 5-column CSV
  • Quoted fields for commas/quotes
  • Proper escape/parse round-trip
Section 09

Bugs We Found & Fixed

Real bugs discovered during development and testing, with the root cause and solution.

Bug: regex.test() + replace() with global flag

Calling regex.test(text) on a /g regex advances lastIndex. The subsequent text.replace(regex, ...) starts mid-string and misses the first match.

Fix: Skip test(), compare replace() result directly

// BEFORE (bug): test() eats the first match if (regex.test(text)) { text = text.replace(regex, rep); } // AFTER (fix): just replace and compare const newText = text.replace(regex, rep); if (newText !== text) { text = newText; }

Bug: \b word boundary fails on $50, (555), v2.0

\b sits between \w and \W. Since $ is already \W, there's no \w→\W boundary before it, so \b\$50\b never matches.

Fix: Use whitespace lookaround for special-char terms

// Detect and use appropriate boundary strategy const hasSpecial = /[^a-zA-Z0-9_]/.test(term); if (hasSpecial) pattern = `(?<=^|\\s)${escaped}(?=\\s|$)`; // whitespace else pattern = `\\b${escaped}\\b`; // word boundary

Bug: MutationObserver fires 42x per page update

Every DOM mutation triggered a full performReplacements() — storage read + recursive DOM walk. On Salesforce Lightning, this meant 40+ wasted executions per content load.

Fix: 150ms debounce on the observer callback

// Batch rapid mutations into one execution let timer = null; const observer = new MutationObserver(() => { clearTimeout(timer); timer = setTimeout(() => performReplacements(), 150); });

Bug: Scope */view/* doesn't match /view

Pattern */view/* compiles to regex .*/view/.* which requires at least one character after /view/. URLs ending with /view or /view?q=1 failed.

Fix: Use */view* (no trailing slash requirement)
